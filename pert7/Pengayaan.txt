Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa perbedaan antara interface dengan class biasa?
2. Mengapa interface tidak memiliki implementasi method di dalamnya?
3. Apa tujuan penggunaan keyword "implements" pada sebuah class?
4. Apakah sebuah class dapat mengimplementasikan lebih dari satu interface? Jelaskan dan berikan contoh sederhana.
5. Jika sebuah class mengimplementasikan interface, apa yang terjadi jika class tersebut tidak mengimplementasikan semua method dari interface tersebut?

jawab
1.Interface adalah struktur yang hanya berisi deklarasi method (kontrak) tanpa implementasi, sedangkan class biasa dapat memiliki atribut dan method dengan implementasi lengkap. Interface digunakan untuk mendefinisikan perilaku yang harus dimiliki class, sementara class biasa digunakan untuk membuat objek dengan fungsi yang sudah jelas.
2.Interface tidak memiliki implementasi method karena fungsinya hanya sebagai aturan atau kontrak perilaku yang harus diikuti oleh class yang mengimplementasikannya. Detail implementasi sepenuhnya diserahkan kepada class turunan agar fleksibel dan sesuai kebutuhan masing-masing.
3.Keyword implements digunakan untuk menunjukkan bahwa sebuah class bersedia mengikuti dan menyediakan implementasi dari seluruh method yang didefinisikan dalam interface. Dengan kata lain, class tersebut menyatakan bahwa ia memenuhi kontrak perilaku dari interface tersebut.
4.Ya, sebuah class dapat mengimplementasikan lebih dari satu interface karena Java mendukung konsep multiple interface. Hal ini memungkinkan sebuah class memiliki banyak perilaku yang didefinisikan dari berbagai interface.Contoh: class Mesin implements Hidupkan, Matikan { }
5.Jika sebuah class mengimplementasikan sebuah interface tetapi tidak menyediakan implementasi untuk seluruh method yang ada di dalam interface, maka class tersebut wajib dideklarasikan sebagai abstract. Jika tidak, Java akan memberikan error karena interface merupakan kontrak yang harus dipenuhi sepenuhnya.
