Setelah anda menyelesaikan kode, tolong jawab pertanyaan dibawah ini ya =


1. Apa tujuan utama adanya interface dalam pemrograman berorientasi objek, dan bagaimana interface membantu menciptakan kontrak perilaku?
2. Mengapa interface dapat memiliki banyak implementasi, dan apa makna konseptual dari kemampuan tersebut dalam desain sistem?
3. Apa perbedaan mendasar antara extends dan implements dalam konteks interface, jika dilihat secara konsep tanpa melihat contoh kode?
4. Apa alasan diperkenalkannya default method dalam interface, dan bagaimana fitur ini memengaruhi keberlangsungan (compatibility) suatu sistem yang sudah berjalan?
5. Bagaimana konsep multiple interface memperluas fleksibilitas desain tanpa menimbulkan masalah kompleksitas yang biasa muncul pada multiple inheritance?

jawab
1.Tujuan utama interface adalah untuk menyediakan kontrak perilaku dalam OOP. Interface mendefinisikan kumpulan method yang harus dimiliki oleh class pengimplementasinya, tanpa menetapkan bagaimana implementasinya. Dengan cara ini, interface memastikan bahwa class yang berbeda tetap konsisten dalam menyediakan kemampuan tertentu meskipun cara kerjanya berbeda.
2.Interface dapat memiliki banyak implementasi karena tidak menentukan detail perilaku, hanya definisi method. Secara konseptual, hal ini memungkinkan berbagai class yang berbeda memiliki kemampuan yang sama namun dengan cara kerja masing-masing. Konsep ini mendukung fleksibilitas desain dan prinsip "many forms" dalam polimorfisme.
3.Secara konsep, extends digunakan untuk mewarisi karakteristik—baik perilaku maupun struktur—dari parent class atau interface. Sementara itu, implements digunakan untuk menyatakan bahwa sebuah class berkomitmen memenuhi seluruh kontrak method yang ditentukan oleh interface. Dengan kata lain: extends berarti mewarisi, sedangkan implements berarti memenuhi.
4.Default method diperkenalkan agar interface dapat berevolusi tanpa merusak kompatibilitas dengan implementasi yang sudah ada. Sebelum fitur ini ada, penambahan method baru pada interface akan memaksa semua class pengimplementasi untuk ikut memperbarui kodenya. Dengan default method, interface dapat menyediakan implementasi bawaan sehingga sistem lama tetap berjalan tanpa perubahan.
5.Konsep multiple interface memberikan fleksibilitas tinggi karena sebuah class dapat mengimplementasikan lebih dari satu interface tanpa membawa kompleksitas yang biasa terjadi pada multiple inheritance. Hal ini dimungkinkan karena interface tidak membawa state (atribut), melainkan hanya kontrak perilaku. Dengan demikian, class dapat “memiliki banyak kemampuan” tanpa mengalami konflik pewarisan struktur atau data.
